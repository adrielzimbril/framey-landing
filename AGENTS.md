# Framey Project Instructions

- **Package Manager**: Always use `pnpm` for installs, scripts, and lockfile updates. Never use `npm` or `yarn`.
- **Design Philosophy**: Framer-like, minimalist, professional, high-fidelity.
- **Visual Rules**:
    - **NO GRADIENTS**: Strictly avoid `bg-gradient-*`, `text-gradient`, CSS linear/radial/conic gradients, and decorative glow fields.
    - **Minimalist Shadows**: Avoid excessive or large shadows. Prefer thin, crisp borders, spacing, and surface contrast.
    - **Bento Grid**: Use strict bento layouts for features, testimonials, pricing, and product simulations.
    - **Color Palette**: Stick to a monochromatic/neutral palette (White, Off-white, Gray, Black). Use one accent color if absolutely necessary, but keep it subtle.
    - **Typography**: Use premium, clean typography (Inter, Geist, or similar).
    - **Product Screens**: Prefer realistic builder simulations (chat, files, canvas, inspector, deploy logs) over generic showcase images.
- **Git**: Commit frequently with descriptive messages. Push to GitHub when significant milestones are reached.
